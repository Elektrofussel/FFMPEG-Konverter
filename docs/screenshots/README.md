# Screenshots Guide / Screenshot-Leitfaden

EN: This folder contains UI screenshots used in the main `README.md`.

DE: Dieser Ordner enthält UI-Screenshots für die `README.md`.

## File Names

Use these exact names:

1. `01_source_media.png`
2. `02_conversion.png`
3. `03_summary.png`
4. `04_progress_log.png`
5. `05_hardware_codecs.png`

## Recommended Resolution

- EN: 1600x900 (or 1920x1080) PNG
- DE: 1600x900 (oder 1920x1080) PNG

## What to Capture

### 01_source_media.png
- EN: Source/target folder section, scan button, file table with multiple files.
- DE: Quell-/Zielordner-Bereich, Scan-Button, Dateitabelle mit mehreren Dateien.

### 02_conversion.png
- EN: Export settings, preset controls, encoder/audio options, AMF/NVENC/QSV related fields.
- DE: Exporteinstellungen, Preset-Steuerung, Encoder-/Audio-Optionen, AMF/NVENC/QSV relevante Felder.

### 03_summary.png
- EN: Summary text, conflict hint, size estimate, start/stop controls.
- DE: Zusammenfassungstext, Konflikthinweis, Größenschätzung, Start-/Stopp-Steuerung.

### 04_progress_log.png
- EN: Running or finished queue with visible global progress and log output.
- DE: Laufende oder fertige Queue mit sichtbarem Gesamtfortschritt und Log-Ausgabe.

### 05_hardware_codecs.png
- EN: Hardware runtime status, generated preset count, codec overview.
- DE: Hardware-Runtime-Status, Anzahl generierter Presets, Codec-Übersicht.

## Capture Checklist

- EN:
  - Use the same UI language as your README examples (or provide both EN/DE sets).
  - Avoid personal/sensitive paths in screenshots.
  - Keep text readable (do not scale too small).
  - Prefer light compression to keep quality.
- DE:
  - Nutze dieselbe UI-Sprache wie in deinen README-Beispielen (oder liefere EN/DE-Sets).
  - Keine persönlichen/sensiblen Pfade im Screenshot.
  - Gute Lesbarkeit sicherstellen (nicht zu klein skalieren).
  - Nur leicht komprimieren, damit die Qualität erhalten bleibt.
